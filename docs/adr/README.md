# docs/adr/ — Architecture Decision Records

Dated records of significant decisions: context, decision, alternatives
rejected, consequences. Numbered (`0001-<slug>.md`), append-only — never
rewritten, only superseded by a later ADR (flip `Status:` and add a pointer).

May stay empty for a long time — that's fine. Write one when a decision is
big enough that future-you (or a teammate) will ask "why is it like this?"

Each record: `Status:` header → Context → Decision → Alternatives rejected →
Consequences (Michael Nygard's ADR format).
